# 애플리케이션 테스트

---

## 프로그램 실행 여부에 따른 테스트

| 구분 | 설명 |
|------|------|
| **정적 테스트** | 프로그램을 실행하지 않고 **명세서나 소스코드 분석** |
| **동적 테스트** | 프로그램을 실행하여 **오류를 찾음** |

---

## 테스트 기반에 따른 테스트

| 구분 | 설명 |
|------|------|
| **명세 기반 테스트** | 사용자의 요구사항에 대한 **명세를 빠짐없이 구현**하는지 테스트 |
| **구조 기반 테스트** | 소프트웨어 **내부의 논리흐름**에 따라 테스트 케이스를 작성하는지 테스트 |
| **경험 기반 테스트** | 유사 소프트웨어나 기술 등에 대한 **테스터의 경험**을 기반으로 테스트 |

---

## 시각에 따른 테스트

| 구분 | 설명 |
|------|------|
| **검증 (Verification) 테스트** | **개발자** 시점에서 제품의 **생산과정** 테스트 |
| **확인 (Validation) 테스트** | **사용자** 시점에서 생산된 제품의 **결과** 테스트 |

---

## 목적에 따른 테스트

| 구분 | 설명 |
|------|------|
| **회복 (Recovery) 테스트** | 여러가지 결함을 줘 실패하도록 한 뒤 **올바르게 복구되는지** |
| **안전 (Security) 테스트** | 시스템 보호도구가 **불법적인 침입으로부터 보호되는지** |
| **강도 (Stress) 테스트** | 과도한 정보량을 주어 과부하시에도 **소프트웨어가 정상 작동하는지** |
| **성능 (Performance) 테스트** | **실시간 성능**이나 **전체적 효율성** 진단 |
| **구조 (Structure) 테스트** | 소프트웨어 **내부의 논리적 경로, 소스코드의 복잡도** 등을 평가 |
| **회귀 (Regression) 테스트** | 소프트웨어의 변경 또는 **수정된 코드에 결함이 없는지** 테스트 |
| **병행 (Parallel) 테스트** | 버전이 다른 소프트웨어에 **동일한 데이터를 입력**하여 결과를 비교하는 테스트 |
