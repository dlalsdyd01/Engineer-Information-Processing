# 오류정정 방식

---

## 후진 오류 정정 (BEC)

> 오류가 발생하면 역방향 채널을 통해 송신측에 재전송을 요구

- Parity, CRC 등을 사용하여 검출, 오류정정은 **자동 반복 요청(ARQ)**으로 수행
- 자동 반복 요청 (ARQ: Automatic Repeat reQuest)

### ARQ 종류

| 종류 | 설명 |
|------|------|
| **정지-대기 ARQ** | ACK(긍정응답), NAK(부정응답) |
| **연속 ARQ** | Go-Back-N ARQ |
| **적응적 ARQ** | 선택적 재전송 ARQ (잘 안씀) |

---

## 패리티 검사 (Parity Check)

- 데이터블록에 **1비트**의 검사비트인 패리티 비트를 추가하여 오류 검출
- 가장 간단한 방식이지만 오류가 **2개 이상** 발생시 검출 불가능
- 오류만 검출가능, **수정 불가능**

---

## 순환 중복 검사 (CRC)

- 프레임 단위로 오류검출을 위한 **다항식코드(FCS)**를 사용해 오류검출
- 동기식 전송에 사용되는 에러검출기법으로, 데이터가 프레임단위로 전송시 사용
- 여러개 오류가 났을때 검출가능, **검출률이 높아 가장 많이 사용된다**

---

## 전진 오류 정정 (FEC)

- 데이터 전송과정에서 발생한 오류를 검출해 **스스로 수정하는 방식**
- **해밍 코드(Hamming Code)**를 사용하여 오류검출하여 수정

---

## 해밍 코드 (Hamming Code)

- 정정 수까지 하여 **자기정정부호**라고도 한다
- 하나의 데이터 단위에 패리티 비트를 추가하여 오류를 검출 후 교정이 가능한 코드
- **2개의 오류를 검출 가능, 1개의 오류 교정**
- 잉여 비트가 많이 필요함
- 전송비트중 **2^n번째**를 오류정정을 하기 위한 패리티 비트로 사용
