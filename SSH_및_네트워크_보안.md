# SSH 및 네트워크 보안

---

## SSH (Secure Shell)

- 다른컴퓨터에 로그인, 원격명령 실행, 파일복사등을 수행할 수 있도록 다양한 기능을 지원하는 프로토콜, 또는 이를 이용한 응용프로그램
- 데이터 암호화 + 강력한 인증 방법으로 보안성이 낮은 네트워크에서도 **안전하게 통신가능**
- 기본적으로 **22번포트** 사용
- 키(key)를 통한 인증방법 사용시 사전에 클라이언트 공개 키를 서버에 등록 해야함

## TCP 접속에 사용되는 서비스별 포트 번호

| 서비스 | 포트 번호 |
|--------|-----------|
| **FTP** | 21 (변경가능) |
| **TELNET** | 23 |
| **SMTP** | 25 |
| **DNS** | 53 |
| **HTTP** | 80 |
| **SSH** | 22 |

## 탬퍼 프루핑 (Tamper Proofing)

- 소프트웨어의 위·변조가 발생한 경우 소프트웨어를 오작동하게하여 **악용을 방지**
- 해시함수, 핑거프린트, 워터마킹등 보안요소를 생성하여 삽입
- **실행코드 난독화, 원본 비교 및 데이터 확인**을 수행하여 소프트웨어 보호

---

## OAuth (Open Authorization, 공개인증)

- 인터넷 어플리케이션에서 사용자 인증에 사용되는 **공개 API**
- 사용자들이 비밀번호를 제공하지 않고 다른 웹사이트 상 자신의 정보에 대해 **접근권한을 부여**할 수 있다
- 2010년 IETF에서 1.0, 2.0 표준이 완료되었다
