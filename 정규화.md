# 정규화 (Normalization)

함수적 종속성 등의 이론을 이용하여 잘못 설계된 관계형 스키마를 더 작은 속성의 세트로 쪼개어 바람직한 스키마로 만들어가는 과정.

정규화는 개념적 설계 단계와 논리적 설계 사이에서 수행한다.

데이터 속성의 중복성을 최소화하여 일치성을 보장하고, 데이터 모델을 단순하게 구성한다.

## 정규화 특징

- 효과적인 검색 알고리즘 생성 가능
- 중복을 배제하여 삽입, 삭제, 갱신 이상 등의 발생 방지
- 데이터 삽입 시 릴레이션을 재구성할 필요를 줄인다
- 데이터베이스 내부 자료의 무결성 유지를 극대화한다

## 이상(Anomaly) 현상

**삽입 이상**: 릴레이션에 데이터를 삽입할 때 의도와 상관없는 값들도 함께 삽입.

**삭제 이상**: 릴레이션에서 튜플을 삭제할 때 의도와 상관없는 값들도 함께 삭제.

**갱신 이상**: 릴레이션에서 튜플에 있는 속성값을 갱신할 때 일부 튜플의 정보만 갱신되어 정보에 모순이 생기는 현상.

## 정규화 과정

```
비정규 릴레이션
  │  도메인이 원자값
  ↓
 1NF
  │  부분적 함수 종속 제거
  ↓
 2NF
  │  이행적 함수 종속 제거
  ↓
 3NF
  │  결정자이면서 후보키가 아닌 것 제거
  ↓
BCNF
  │  다치 종속 제거
  ↓
 4NF
  │  조인 종속성 이용
  ↓
 5NF
```

| 정규형 | 조건 |
|---|---|
| 1NF | 도메인이 원자값 |
| 2NF | 부분적 함수 종속 제거 |
| 3NF | 이행적 함수 종속 제거 |
| BCNF | 결정자이면서 후보키가 아닌 것 제거 |
| 4NF | 다치 종속 제거 |
| 5NF | 조인 종속성 이용 |
