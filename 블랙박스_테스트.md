# 블랙박스 테스트

> 소프트웨어가 수행할 특정 기능을 알기 위해서 **각 기능**이 완전히 작동되는 것을 입증하는 테스트로, **기능테스트**라고도 함

- 사용자의 요구사항 **명세**를 보며 테스트 함
- 주로 **구현된 기능**을 테스트 함
- 소프트웨어 **인터페이스**를 통해 실시됨

---

## 블랙박스 테스트 기법

### 동치분할 검사 (= 동등 분할 기법)
- 프로그램 입력조건에 타당한 입력자료와 타당하지 않는 입력자료의 개수를 균등하게 하여 테스트 케이스를 정하고, 해당 입력 자료에 맞는 결과가 출력되는지 확인하는 기법

### 경계값 분석
- 입력조건의 중간값보다 **경계값**에서 오류가 발생할 확률이 높다는 점을 이용하여 입력조건의 경계값을 검사하는 기법

### 원인-효과 그래프 검사
- 입력 데이터 간의 **관계**가 출력에 미치는 상황을 체계적으로 분석한 다음 효율성이 높은 테스트 케이스를 선정하여 검사

### 오류 예측 검사
- 과거의 경험이나 확인자의 **감각**으로 테스트하는 기법

### 비교 검사
- 여러 버전의 프로그램에 동일한 테스트 자료를 제공하여 동일한 결과가 출력되는지 테스트하는 기법
